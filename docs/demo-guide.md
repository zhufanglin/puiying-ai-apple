## 2. 演示场景

### 场景 A：奖状颁发全流程

**入口**: 左侧导航 → Awards 或 http://localhost:3000/dashboard/apple/awards

**步骤**:
1. 查看奖状列表（显示 3 条已发布奖状）
2. 点击奖状查看详情（含获奖学生列表及证书编号）
3. 查看奖状模板（3 Good Students / Service / Academic）
4. 查看统计数据（总奖状数、获奖学生数、奖学金申请数）

**预期结果**: 奖状列表、详情、模板、统计均正确显示数据。

---

### 场景 B：财务收支全流程

**入口**: 左侧导航 → Finance 或 http://localhost:3000/dashboard/apple/finance

**步骤**:
1. 查看收入 Tab（2 笔：Building Fund ,000 + Tuition ,000）
2. 切换到支出 Tab（1 笔：Equipment ,000，状态 Pending）
3. 查看报价单 Tab（2 笔：Tech Supplies 被选中、Office Pro 未选中）
4. 点击"上传收据"体验 OCR 上传对话框

**预期结果**: 三个 Tab 数据正确，报价单显示选中/未选中状态。

---

### 场景 C：学生事务全流程

**入口**: 左侧导航 → Students 或 http://localhost:3000/dashboard/apple/students

**步骤**:
1. 查看学生总览页（5 名在学学生）
2. 查看统计数据卡片（在学人数/考勤异常/待办事项）
3. 搜索/筛选学生
4. 点击学生查看详情页（基本信息、考勤记录、成绩）

**预期结果**: 学生列表、详情、统计数据均正确显示。

---

### 场景 D：资产盘点

**入口**: 左侧导航 → Assets 或 http://localhost:3000/dashboard/apple/assets

**步骤**:
1. 查看资产列表（3 项：Projector / Piano / Desktop）
2. 查看资产状态（2 Active + 1 Written Off）
3. 查看资产注销记录

**预期结果**: 资产列表按状态正确展示。

## 3. 演示要点

| 要点 | 说明 |
|------|------|
| 权限区分 | 用不同账号登录展示不同模块的可见性 |
| 响应式 UI | 缩小浏览器窗口查看卡片布局自适应 |
| 空状态 | 删除数据后查看 EmptyState 组件显示 |
| 错误处理 | 输入无效数据查看前端校验反馈 |
| 加载状态 | 观察 DataTable 组件的 loading 状态 |

---

## 4. 常见演示问题

| 问题 | 处理方式 |
|------|---------|
| 页面空白 | 检查 Network Tab，确认 API 是否可达 |
| 数据不显示 | 确认已运行种子脚本，数据库中有数据 |
| 登录失败 | 确认 DATABASE_URL 指向正确的数据库文件 |
| 前端样式错乱 | 清空浏览器缓存，重新加载 |
